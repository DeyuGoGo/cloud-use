# 04 — 美術與製作

## 一個原則（這次學到的）
立繪能做到的「特色」有限——再漂亮的圖也只是「一個好看的人」。**角色讓人記得，是靠他在劇情裡怎麼想、怎麼做，不是那張臉。**
- 美術別追求「通用美女」。十幾個角色不能長同一張臉——先用年齡、體型、氣質、具體特徵（疲憊感、痣、不對稱、不完美）把人分開。
- 真正的人味在 [02 角色](02-角色.md) 和劇情，立繪只是底。

## 兩條產線（實測後分工）

**SFW（立繪、背景、宣傳）→ codex (gpt-image)**
- 風格可指定：韓系光澤插畫（金亨泰／棕色塵埃那種）、低飽和夜景、運動裝。品質高、一句話到位、背景強（會自己擺台北 101）。
- 同角色一致性：靠 `-i` 餵參考圖維持（先定一張設定圖，之後都餵回去；實測「同一個人換姿勢」臉守得住，有輕微飄移但認得出）。
- 限制：**不能 R18**。

**R18、以及需要嚴格鎖臉 → ComfyUI**
- 風格：K2／Anima 電影感（低飽和夜景、waiANIMA），用 LoRA／固定 seed 鎖臉。
- 一句話：**SFW 跟概念圖用 codex；R18 跟嚴格一致用 ComfyUI。**

## codex 指令（已驗證可用）
```powershell
$prompt = @'
<prompt 寫這裡；風格要明寫 NOT photorealistic，否則它預設走寫實>
'@
$prompt | codex exec --skip-git-repo-check -s workspace-write -C "<工作目錄>" [-i "<參考圖>"]
```
- prompt 走 **stdin**（別放位置參數——`-i, --image` 是可變長度，會把後面的 prompt 當成圖檔吃掉）。
- codex 內建 image_gen，**不需 API key**；先存 `~/.codex/generated_images/`，再 copy 到工作目錄。

## ComfyUI 注意事項
詳見記憶 `comfyui-image-pipeline`：comfy-aimdo 要 **0.4.9**、用 **headless** 跑（開 Desktop 會把它還原成壞的 0.3.0）、API 在 **8001**、腳本在 `ComfyUI-Knowhow/scripts/`。
