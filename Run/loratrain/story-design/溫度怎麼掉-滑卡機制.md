# 溫度怎麼掉——「玩家感覺到、卻沒人告訴他」在滑卡形式裡長什麼樣子

> 這份文件回答整個案子的命門設計題：核心引擎是「正當的自利 → 漸行漸遠」，立翔越爬越麻、全程理直氣壯、找不到自己哪裡做錯；真正的情緒在**錯位**——玩家看見立翔看不見的溫度下降。問題是：在 VN×Reigns 左右滑卡的形式裡，這件事具體怎麼做？
>
> 產出方式：6 個角度並行發想 30 個機制 → 3 位對抗評審（引擎忠誠度 / 可玩性 / 鐵律守門「抓作弊」）交叉打分。下面是篩過的結論。
> 放在 `loratrain/` 底下避免 git 衝突。動到結構處用 🟡 標、等得瑜確認。

---

## 一句話答案

> **溫度掉 = 玩家自己做的「減法」，而且最好是用「玩家自己的手」做的。**

遊戲一個字都不能說「他變了 / 大家疏遠他了」。溫度活在三個地方，玩家在那裡自己算出那一度涼，立翔全程無感。

---

## 溫度住在滑卡的三個地方（依優先序）

### ① 玩家的手勢本身（脊椎，最該先做）
同一個「向右滑＝對自己好、被看見」的手勢，玩家在序章被訓練成「右＝爽事」；後期**同樣的手感、同樣的暖色 tint、同樣的 icon**，這一滑卻是把怡君關靜音。是「玩家自己的手」下的刀，比任何旁白狠十倍。
→ 同時解決了我先前擔心的「主角太被動、玩家沒在開車」：**把溫度下降做成核心動作本身**，玩家的手一直有事做、且事後會「想再滑一次去確認/補救」——這正是耐玩鉤子。

### ② 同一張卡/動作的第二次（callback 減法）
母題卡前期演一次、後期同構圖再演一次，只改一兩個物件或幾個字。玩家記得第一次寫什麼，拿這次去減，差值就是涼意。立翔每次都只在做「當下這一次」，沒有對照組，所以無感；玩家有對照組，替他痛。

### ③ 餘光與缺席（立翔看不到的角落）
同一張卡上資訊不對稱：玩家餘光吃進對方一個小動作（凱文「正在輸入…」打了又刪），立翔盯著讚數永遠看不到。或把「代價畫面」延遲幾場，等立翔最高昂時才回放帳單。

## 七個值得做的機制（評審交叉背書順序）

> 標註：🟢引擎 / 🔵可玩性 / 🟠鐵律守門，分數 1–5。

### 1.〔脊椎〕回收同一個手勢 + 鏡像滑卡　🟢4 🔵5(該最先做) 🟠4
把序章某張**真的給過選項**的滑卡，後期用同一句問句、同一組左右標籤再出一次（玩家肌肉記憶會自動往他習慣那邊滑）。功能不可反（←clean／→seen），但這次右滑的反應 beat 演的是溫度掉那一刻——**不是因為他選錯，是因為他選了跟上次一模一樣「正確」的那邊。**
- **例**：後期新建一張刻意做得像 01「手機一直在響——理它嗎？」的孿生卡。問句「怡君傳了一長串，問你今天怎樣——現在回嗎？」←先擺著／→先回現場這群（合照在等你）。玩家照習慣右滑。右滑 fb 只寫立翔：「你被叫去站中間，手機蓋在桌上。很久以後翻回來，那串字你沒往上滑看，回了兩個字：哈哈。對話框安靜了。她沒有再傳。」
- **鐵律守則**：右滑永遠 seen+、**絕不偷偷變成懲罰**（玩家一察覺「右滑開始扣分」就毀了「自利單看都對」）。fb 只演怡君的安靜，**絕不演立翔的損失或懊悔**——立翔該爽的還是爽。
- 🟡 這條是脊椎，「回收手勢」和「鏡像滑卡」其實是同一招，合併成一條主軸做。

### 2.〔脊椎〕靜默成本：放手後那半秒「什麼都沒發生」　🔵5 幾乎零成本
引擎本來 fb 為空就直接切下一場。玩家整局被訓練成「滑完一定有一句暖的/有趣的回饋」。在**少數決定性的自利滑動**上，故意把 beat 抽掉或縮到極短，畫面比平常更快、更安靜地切走。那個「冷場」本身＝玩家版的「半秒的涼」，沒有一個字說溫度掉了。
- **守則**：只在 2–3 個關鍵拍用；不要真的空白（會被當當機），留一句極短到近乎冷場的動作 beat。

### 3. 那盞燈：開場—收尾包夾　🟢5 🟠5
用「那盞燈」當整章書擋。章首 witness：立翔出門「手按開關上——這次沒停，順手關了」（已經沒有「萬一有人回來」這回事，但不點破）。章尾 witness 三拍收：「回到巷口，窗戶是暗的」→「進門伸手要開燈，才想起出門時自己關了」→「屋裡沒有為誰亮著的那盞。」對照序章 07 留燈、13 燈還亮。
- **守則**：章尾只能停在物理事實「燈是暗的、是你關的」。**禁止**寫「你已經不期待有人回來了」。

### 4. 反向回收：連「做對的選擇」都救不回　🟢5 🟠4（最講透命門、最難寫）
某些母題卡早期左滑（乾淨/自己）真的什麼都不發生、fb 中性；後期同一張卡左滑，把**早期那句中性 fb 幾乎原樣回收**，但這次它讀起來變涼，因為周遭人已經少了。
- **例**：怡君在堤防超過你（03→11）。早期左滑「到這裡就好」fb：「你停在護欄邊喘，她的頭燈在前面一晃一晃。明天她還會在。」後期立翔難得回河邊跑，左滑 fb 幾乎同句：「你停在護欄邊喘。**前面沒有頭燈了。**」——怡君沒翻臉、沒那一刀，她只是不再來這條河了（因為立翔不在了），而立翔回來時剛好錯過。
- **為何最強**：立翔這次選的是最乾淨、最不傷人的那邊，結果還是涼了——證明溫度掉根本不是哪個壞選擇造成的，是累積的缺席。**連做對都救不回，因為傷的是感受不是利益。**
- **守則**：「前面沒有頭燈了」差一個字就滑進自憐/旁白。**必過 Codex 二審。**

### 5. Ann 那罐：頓半秒越來越長　🟠5（show 不 tell 純度最高）🔵3（佐料非主菜）
同一個「超商前 Ann 多買一罐」場景多次回收，每次把她「多買前頓半秒」用文字節奏拉長一格，且每句 fb 後半把立翔釘在「正在看自己的讚」。沒頓 → 停一下 → 停得有點久 → 那罐自己留著沒發。
- **終局 witness**：輪到 Ann 生日，她照樣買一整袋一罐一罐發，發完自己靠牆開一罐。**只擺事實，禁止寫「沒人記得她生日 / 她好寂寞」。**

### 6. 已讀數據化漂移：剛剛→昨天，人名→機器人　🟢4 🟠4
「群組 99+ 滑到底」回聲卡（01→13→後期）。每次同一個滑到底的動作，只改數據：最後 @ 你的人，從具體人名（「立翔等等到喔！」時間戳「剛剛」）→ 隔著 30 多則別人 @ 別人 →「昨天」→ 最後變成系統機器人「@所有人 三週年大合照」。立翔照例已讀、把手機翻過來蓋桌上。玩家從時間戳和 @ 對象的退化讀出「沒人在特地找你了」。
- 全用人話呈現（剛剛/昨天、某某/機器人），**零數字**。

### 7. 看不見對面的滑動：餘光看見、立翔看不見　🟢4 🟠4（分寸最窄）
同一張卡的構圖：前景是立翔要滑的主體（手指停在亮著的「分享」鈕），卡面角落極小一行 IG 通知「kevin 對你的限動…正在輸入…」灰掉、消失（凱文打了又刪 motif）。右滑分享後 fb 只寫立翔的爽：「限動發出去，十秒內第一個讚是 Ray。」完全不提凱文那行字消失了。
- **守則**：那行字要小、要快、要在玩家視線自然掃過滑動軌跡時才撞見；太明顯＝替玩家下結論，太隱晦＝整個漏看。每場至多一個。

## 守門鐵律（一個範例講清「作弊 vs 乾淨」）

同一個 Ray「記得有用的、忘掉沒用的」母題，有兩種寫法，評審判一生一死：

- ❌ **作弊（被淘汰）**：fb 寫「香菜那件事他大概早忘了，但這個他記得牢牢的。」——把該由玩家做的減法**寫成了句子**，等於旁白下結論。
- ✅ **乾淨（滿分）**：Ray 早期忘了立翔說過不吃香菜（下次團購還是幫他點加香菜的）；後期立翔有流量了，菜端上來有香菜，**Ray 順手把那盤香菜推到立翔面前「多吃點」**。沒有一個字解釋，玩家自己回頭涼。

> **鐵律 = 把結論留給玩家。任何「替玩家說破」的句子、字幕、數字、進度條，一律砍。**

## 淘汰名單（別浪費力氣做的）

- **全局調色類**（金框暖珠褪冷白 / 卡背結霜）：可玩性與鐵律評審愛（零字零數字），但**引擎評審判死**——純氛圍、沒有對照錨點，玩家會讀成「遊戲整體變冷了」而非「我傷了誰」，等於視覺版的旁白下結論。**結論：不能當機制，最多當「已錨定機制」底下極輕的氛圍放大層，且要 4–5 階做到無感漸變。**
- **orb 右側遲鈍**：把「被看見」那側做成可感知的衰減，逼近「右滑＝被懲罰」，破壞「自利單看都對」，且讀成「立翔被虧待」（偏離「傷的是別人感受」）。**捨。**
- **延遲回填 witness（回憶殺）**：情緒美，但在滑卡快節奏裡是踩剎車、近乎替玩家收尾。**最多全局 2 次，重武器，不進核心機制。**
- **消失的選項（動態增刪 left/right）**：分支爆炸難測，且玩家多半沒注意到就失效、或被讀成 bug。**用「同手勢不同後果」(機制1) 取代。**
- **群組向下滑手勢**：引擎只支援水平滑，做不起來。退回純 tap 就失去賣點。**捨。**
- **老吳繫鞋帶句點改由立翔繫**：概念最深（引擎 5 分），但**可玩性判死**——純 witness、零互動、是減速帶。**留作極後期一次性收尾，不進耐玩機制前段**；且「你想起凱文當初也沒記住你」這句要砍到只剩「鞋帶本來就好好的」。

## 對「可玩性那 1.5 分」的回答

先前我給主軸 8.5/10，扣的 1.5 在「軸跟可玩性天生打架（軸靠慢、靠看不見；滑卡靠即時回饋）」。這套機制的解法是：

> **把溫度下降做成「玩家最常做、最爽的那個右滑手勢本身」(機制 1+2)。** 玩家的手一直有事做、有立即觸感，且「我親手做的」會逼他想再滑一次去確認/補救——溫度掉不再是慢吞吞的文學閱讀，而是核心動作的一部分。

witness-heavy 的機制（老吳、回憶殺）才是會讓節奏熄火的，所以它們降級成「極後期一次性收尾」，不當引擎。

## 建議的開發順序

1. **先做脊椎**：機制 1（鏡像滑卡/回收手勢）＋ 機制 2（靜默成本）。這兩個用既有引擎、幾乎零新系統，且直接驗證「玩家會不會自己痛」這個賭注。先拿怡君「你呢」那條做一張可玩切片。
2. **再鋪 callback 母題**：那盞燈 bookend（機制 3）、怡君頭燈反向回收（機制 4）、群組數據漂移（機制 6）。
3. **佐料**：Ann 那罐（機制 5）、餘光凱文（機制 7）少量點綴。
4. **收尾重武器**：老吳繫鞋帶、回憶殺，極後期各一次。
